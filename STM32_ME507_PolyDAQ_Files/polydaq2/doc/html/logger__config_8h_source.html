116 /** @brief Parser for simple data logger configuration files.
117  * @details This class parses simple configuration files, kept on SD cards, for
118  * Swoop or PolyDAQ 2 data loggers. Data is recorded by taking measurements
119  * from the specified channels, at the given rate in milliseconds per sample.
120  * The data is scaled by a simple linear gain and offset, then written onto
121  * the SD card.
122  *
123  * The configuration file can contain the following:
124  * \li Comments which describe what's in the file. A comment begins with a
125  * @c "#" and lasts to the end of a line.
126  * \li Empty lines, which are ignored.
127  * \li Data lines which begin with a single letter followed by a colon.
128  * Valid data lines include:
129  *
130  * | Letter | Meaning |
131  * |:------:|:--------|
132  * | H | Header line showing type of PolyDAQ (not used) |
133  * | B | Baud rate for serial interface to GUI (not used) |
134  * | T | Time per data sample, in milliseconds |
135  * | C | Channel configuration line, as detailed below |
136  *
137  * Channel configuration lines look like the following:
138  * @code
139  * C: 9, 1.0, 0.0, "Cow Strain"
140  * @endcode
141  * The items in the channel configuration line are as follows:
142  *
143  * | | |
144  * |:---:|:---------|
145  * | @c 9 | The single-letter command to read data, such as @c 5 or @c X |
146  * | @c 1.0 | Gain by which raw data is multiplied before being saved |
147  * | @c 0.0 | Offset added to data after gain and before data is saved |
148  * | @c "Cow Strain" | Channel name written to data file as column header |
149  */
150
