Detailed Description

Parser for simple data logger configuration files.

This class parses simple configuration files, kept on SD cards, for Swoop or PolyDAQ 2 data loggers. Data is recorded by taking measurements from the specified channels, at the given rate in milliseconds per sample. The data is scaled by a simple linear gain and offset, then written onto the SD card.

The configuration file can contain the following:

  • Comments which describe what's in the file. A comment begins with a "#" and lasts to the end of a line.
  • Empty lines, which are ignored.
  • Data lines which begin with a single letter followed by a colon. Valid data lines include:
Letter Meaning
H Header line showing type of PolyDAQ (not used)
B Baud rate for serial interface to GUI (not used)
T Time per data sample, in milliseconds
C Channel configuration line, as detailed below

Channel configuration lines look like the following:

C: 9, 1.0, 0.0, "Cow Strain"

The items in the channel configuration line are as follows:

9 The single-letter command to read data, such as 5 or X
1.0 Gain by which raw data is multiplied before being saved
0.0 Offset added to data after gain and before data is saved
"Cow Strain" Channel name written to data file as column header

uint16_t logger_config::ticks_per_reading
protected

Number of RTOS ticks per sample set.

The RTOS timer which controls data recording runs much faster than we usually save data; data is only saved once in every this-many RTOS timer ticks. With an ordinary RTOS timer setting of one tick per millisecond, data is taken once in every this many milliseconds.
